Refactor Report
Project: KNN vs Decision Tree Market Sales Kocaeli

Main Change
-----------
The feature vector was changed to match the project requirement better.

Old features:
- Gender
- Brand
- Amount
- LineNet

New features:
- Age
- Gender
- City
- LineNetTotal

The target label is still CATEGORY_NAME1.

Important dataset note:
The current Excel file in this folder does not contain AGE or CITY columns.
DataLoader now supports AGE/YAS and CITY/CITY_NAME/SEHIR/IL columns when they
exist. If the current file does not have them, age is treated as unknown and
city is set to UNKNOWN so the program still runs.


1. src/main/java/model/SaleRecord.java
--------------------------------------
What changed:
- Removed brand, amount, and lineNet fields.
- Added age, city, and lineNetTotal fields.
- Kept clientCode, gender, and categoryName1.

Why it changed:
- The raw data model should match the fields used by the classifier.
- The project requirement focuses on age, gender, city, and spending behavior.

How to explain:
"SaleRecord stores one raw row from the dataset. It now keeps the fields needed
for the required prediction task: age, gender, city, and LineNetTotal."

How to test:
- Load the dataset.
- If AGE and CITY columns exist, check that they are read.
- If they do not exist, the app should still load using safe defaults.


2. src/main/java/model/ProcessedRecord.java
-------------------------------------------
What changed:
- Removed all comments.
- Kept the same simple structure: features, label, original category name.

Why it changed:
- Classifier code still needs a numeric feature vector and an integer label.

How to explain:
"ProcessedRecord is the numeric version of SaleRecord after preprocessing."

How to test:
- Run either classifier and confirm prediction still works.


3. src/main/java/data/DataLoader.java
-------------------------------------
What changed:
- Reads AGE/YAS if present.
- Reads CITY/CITY_NAME/SEHIR/IL if present.
- Reads LINENETTOTAL instead of LINENET.
- No longer reads Brand or Amount for features.
- Removed all comments.

Why it changed:
- The project requirement asks for age, gender, city, and spending behavior.
- LineNetTotal is the spending feature now.

How to explain:
"DataLoader only reads the Excel file and creates SaleRecord objects. It maps
the needed spreadsheet columns into the raw data model."

How to test:
- Run the app and load the Excel file.
- Check loaded/skipped row counts.
- Confirm no loading error occurs.


4. src/main/java/data/PreProcessor.java
---------------------------------------
What changed:
- Feature vector is now:
  [0] Age
  [1] Gender
  [2] City
  [3] LineNetTotal
- City is label-encoded and normalized.
- Age and LineNetTotal are min-max normalized.
- Unknown age becomes 0.5.
- Unknown city becomes 0.0.
- Removed brand encoder.
- Removed all comments.

Why it changed:
- The classifier needs numeric features.
- Age and spending are numeric, while gender and city are categorical.

How to explain:
"PreProcessor converts raw values into numeric values. It fits encoders and
normalization values using training data only, then transforms train and test
records with the same mappings."

How to test:
- Load data.
- Run KNN and Decision Tree.
- Confirm the UI lists Age, Gender, City, and LineNetTotal as features.


5. src/main/java/classifier/KNNClassifier.java
----------------------------------------------
What changed:
- Removed all comments.
- Kept the same KNN algorithm.
- Some local variable names are simpler and include a little Turkish, such as
  komsular, toplam, and fark.

Why it changed:
- KNN does not need to know what each feature means. It only uses the numeric
  feature array.

How to explain:
"KNN stores training data, calculates distances to all records, sorts nearest
neighbors, and returns the majority class."

How to test:
- Select KNN.
- Run with K = 5.
- Confirm accuracy and confusion matrix appear.


6. src/main/java/classifier/DecisionTreeClassifier.java
-------------------------------------------------------

How to test:
